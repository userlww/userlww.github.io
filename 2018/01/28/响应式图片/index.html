响应式图片

2018-01-28 • 分类: 前端, CSS • 标签: CSS, 响应式图片

写在前面

  响应式图片是响应式网站开发过程中必不可少的一部分，我们可以通过使用max-width来让图片自适应，但是这样做的的话，我们必须提前制作好一张分辨率足够高的图片，如果我们不是真的需要那么大的图片，那么这样做不得不说是一种浪费，太过占用带宽会影响到用户体验，所以我们选择用其他的方式来实现响应式的图片，下面我来介绍一下我用过的响应式图片的解决方案。

响应式图片的解决方案

通过创建新的HTML元素来实现

<img srcset="img/index-2.jpg 320w,
img/index-3.jpg 480w,
img/index-4.jpg 800w
sizes="(max-width: 320px) 280px,(max-width: 480px) 440px"
src="img/index-1.jpg" alt"初始的图片" width="300px" height="300px">

  这段代码使用img元素的srcset属性和size属性在不同屏幕大小时显示不同的图片，srcset属性值是由三部分组成

  1. 图片的URL

  2. 一个空格

  3. 图片的固有宽度，用w表示
    多个图片之间用逗号隔开；size属性定义了一组媒体条件，(max-width: 320px)表示的媒体条件是当屏幕宽度等于或小于320px，后面的320px则表示当媒体条件为真时的图片将填充的槽的宽度
      有了这些属性之后，浏览器的执行顺序是

  4. 查看设备的宽度

  5. 检查sizes列表中哪一个媒体条件为真

  6. 查看该媒体查询被给予的槽的大小，加载srcset列表中引用的最接近所选的槽的大小的图片。

  那么在我们的例子中，如果屏幕大小为480px时，480px的图像会被加载，因为480px最接近440px的槽值。
